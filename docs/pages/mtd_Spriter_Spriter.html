Spriter.Spriter( )

Description

A sprite animator feature. Given the sprite image, total number of tiles, number of tiles per line and tile dimensions, this feature produces an animation using FPS variable as the FPS rate.

Compatibility

Prototype

Spriter(DOMObjId:String, tileWidth:Number, tileHeight:Number, spriteImageFile:String, tilesPerLine:Number, totalTiles:Number, loops:Number, fromFrame:Number, keepFromFrame:Boolean)

Parameters

Parameter Type Description
DOMObjIdStringThe DOM Element id
tileWidthNumberThe width in pixels of the tile
tileHeightNumberThe height in pixels of the tile
spriteImageFileStringThe url to the sprite image
tilesPerLineNumberThe number of tiles (frames) per line
totalTilesNumberThe total number of tiles (frames)
loopsNumberThe number of times the animation will loop. For looping infinitely, one can use -1
fromFrameNumberThe frame offset to start the animation
keepFromFrameBooleanIf true, when the animation loops, it starts from fromFrame. Otherwise, it starts from the beginning

Examples

Example 1 

//Creating a 64x64 animation area from a sprite image with 16 tiles that loops infinitely and begin always from frame 10
new Spriter("spritePlace", 64, 64, "spriteImage.png", 4, 16, -1, 10, true);
